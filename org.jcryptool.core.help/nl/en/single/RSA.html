RSA

RSA (Rivest-Shamir-Adleman) is one of the first public-key cryptosystems and it is widely used for encryption and signing. A good general overview can be found in Wikipedia [1] and in chapter 4 of the CrypTool Book [2].

This dialog allows to encrypt and decrypt with RSA using a key from the JCT keystore (keys created here are built with the crypto library FlexiProvider, which only supports big key lengths in order to achieve practical relevance).

In JCT there are further places dealing with RSA:

  • In the Default Perspective there are the following two Visualizations:
    • Visuals → RSA Cryptosystem, which explains the RSA method in detail and also allows smaller key sizes for didactical purposes,
    • Visuals → Extended RSA Cryptosystem, which shows RSA in a broader context: Each communication partner (identity) needs a key pair, the public key has to be exchanged, different keys are used to encrypt (sender) and to decrypt (receiver) messages. In addition the RSA variant "multi-prime RSA" can be applied with small primes.
  • In the Algorithm Perspective in the "Algorithms" tab:
    Below "Asymmetric Block Ciphers" you can find three RSA variants: RSA, RbRSA, and RprimeRSA (they all use big key lengths).
    Similarily, there are RSA variants below "Signatures".


Remark

There are several RSA variants, some of them are offered in the Algorithm Perspective:

  • RprimeRSA means rebalanced multi-prime RSA.
  • RbRSA is another rebalanced RSA variant.
  • MeRSA means multi-exponent RSA used for a special signature scheme by Mihir Bellare and Phillip Rogaway.
  • MpRSA means multi-prime RSA used in the OAEP (optimal asymmetric encryption padding).

More details can be found e.g. in [3] and [4].
