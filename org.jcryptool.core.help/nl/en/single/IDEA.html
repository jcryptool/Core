IDEA

Description

IDEA is a symmetric encryption algorithm which operates on blocks of length 64 bits (= 8 bytes). The length of the key is 128 bits. It was developed in 1990 and patented. The patent expired in 2011.

IDEA was used in PGP (Pretty Good Privacy) and discussed as a successor of DES.

Operation of the dialog

You can just start encrypting the content of the active editor by clicking the button Finish (so you use the default settings). You are then asked for the key protection password, which is "1234" for the pre-selected user "Alice". The result is written in a new hexeditor window of JCT.

In JCT, the key length is determined by the selected key. The dialog in JCT offers the following parameters:

  • for block-chaining: the five modes ECB, CBC, OFB, CFB, and CTR
  • for padding: the three schemes PKCS#5, 01-00 padding, and no padding.

Pressing the long button Create a new (IDEA) key in the keystore, opens a dialog called "New symmetric key". You can create a new key either for an existing user like "Alice Whitehead", or better for a new user like yourself (by just entering a user name in the user field and overwriting "Alice"). You will be asked with which password this new key should be protected. After the new key is stored in the keystore, you can use it again later (without having to create a new key first).

After clicking on Finish you will return to the "IDEA — encryption" dialog, which now lists the newly created key in the "Keystore" grouping. Here you also have to click the button Finish to write the result in a new hexeditor window.
