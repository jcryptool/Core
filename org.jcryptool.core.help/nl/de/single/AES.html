AES

Beschreibung

Der AES-Verschlüsselungs-Algorithmus arbeitet mit einer Blocklänge von 128 Bit und einer variablen Schlüssellänge von 128, 192 und 256 Bit. AES ist die standardisierte Untermenge des zugrundeliegenden Rijndael-Algorithmus (Rijndael bietet eine Blockgröße mit variabler Länge und mehr verschiedene Schlüssellängen).

Für einen 128-Bit-Schlüssel gibt es ungefähr 340.000.000.000.000.000.000.000.000.000.000.000.000.000 = 34*10^37 mögliche Schlüssel. Rijndael wurde von den belgischen Kryptographen Joan Daemen und Vincent Rijmen entwickelt.

AES ist der patentfreie Standard für moderne Blockverschlüsselungen zum Schutz von Langzeitdaten. Er wurde 2001 vom NIST als FIPS-Standard festgelegt und wurde zum Nachfolger des 1977 eingeführten, veralteten Data Encryption Standard (DES) ernannt.

Anmerkung: Um den Algorithmus zu verstehen, ist Wikipedia ein guter Ausgangspunkt, der auf die einschlägige Literatur verweist. Darüber hinaus gibt es gute Animationen des AES-Innenlebens von Zabala (siehe CrypTool 1) und einen berühmten Cartoon mit dem Titel "A Stick Figure Guide to the Advanced Encryption Standard (AES)" von Jeff Moser, der im September 2009 veröffentlicht wurde und den AES-Algorithmus in 4 Akten erklärt.

Bedienung des Dialogs

Sie können den Inhalts des aktiven Editors direkt verschlüsseln, indem Sie auf die Schaltfläche Fertigstellen klicken – damit verwenden Sie die Standardeinstellungen. Sie werden dann nach dem Passwort zum Schutz des Schlüssels gefragt: Für den voreingestellten Benutzer "Alice" ist das "1234". Das Ergebnis wird in ein neues Hexeditor-Fenster von JCT geschrieben.

Bei JCT wird die Schlüssellänge durch den gewählten Schlüssel bestimmt. Der Dialog in JCT bietet die folgenden Parameter an:

  • für die Blockverkettung: die fünf Modi EZB, CBC, OFB, CFB und CTR
  • für das Padding: die drei Schemata PKCS#5, 01-00 Padding und kein Padding.

Drücken der langen Taste Einen neuen (AES-)Schlüssel im Schlüsselspeicher erzeugen, öffnet eine Dialogbox namens "Neuer symmetrischer Schlüssel". Sie können einen neuen Schlüssel entweder für einen bestehenden Benutzer wie "Alice Whitehead" oder besser für einen neuen Benutzer wie Sie selbst erzeugen (indem Sie einfach einen Benutzernamen in das Benutzerfeld eingeben und "Alice" überschreiben). Sie werden gefragt, mit welchem Passwort dieser neue Schlüssel geschützt werden soll. Nachdem der neue Schlüssel in den Schlüsselspeicher gespeichert wurde, können Sie ihn später wieder verwenden (ohne vorher einen neuen Schlüssel erstellen zu müssen).

Nach dem Klicken auf Fertigstellen kehren Sie zum Dialog "AES — Verschlüsselung" zurück, der nun den neu erzeugten Schlüssel in der Gruppierung "Schlüsselspeicher" auflistet. Hier müssen Sie dann auch noch auf Fertigstellen klicken, damit das Ergebnis in ein neues Hexeditor-Fenster geschrieben wird.
